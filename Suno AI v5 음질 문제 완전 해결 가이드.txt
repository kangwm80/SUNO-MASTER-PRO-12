【왜 이런 문제가 생기는가?】 — Suno v5 AI 작동 원리 이해
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Suno는 "화이트 노이즈에서 시작해 정제해가는" 방식으로 음악을 생성한다.
→ 즉, 모든 AI 음악에는 어느 정도의 노이즈 잔재가 존재.
→ 이것이 쉬머(Shimmer), 아티팩트, 배경 노이즈의 근본 원인.

추가로:
· AI는 학습 데이터 패턴을 확률적으로 조합한다.
· 프롬프트가 모호하면 AI가 잘못된 확률 경로로 진입.
· 긴 생성(확장)일수록 누적 오류(Drift)가 증가.
· 특정 감정 단어(!, Powerful, High Energy)는 보컬을 과열시킨다.

핵심 원칙:
  "문제의 80%는 프롬프트에서 예방할 수 있다.
   나머지 20%는 사후 처리(Post-Processing)로 해결한다."

【문제 1】 보컬 괴기음 / 아티팩트 / 환청 (Voice Artifacts)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

증상:
  · 갑자기 로봇 같은 보컬 소리가 섞임
  · 의미 없는 음절 또는 없는 단어를 부름 (Voice Hallucination)
  · 보컬이 갑자기 음이 어긋나거나 뭉개짐
  · 특정 음절이 글리치처럼 반복되거나 건너뜀
  · 메탈릭(금속성) 찰랑거리는 소리 (Metallic Tail)

────────────────────────────────────────────────────────────────────
■ 원인 분석
────────────────────────────────────────────────────────────────────

  원인 A: 프롬프트 신호 충돌
    → 너무 많은 장르 태그, 상충되는 스타일 지시.
    → AI가 어떤 보컬 경로로 갈지 판단 못 함 → 확률 오류 발생.

  원인 B: 가사 음절 밀도 과부하
    → 한 박자에 너무 많은 음절을 넣으면 AI가 처리 실패.
    → 보컬이 뭉개지거나 글리치가 생김.

  원인 C: AI의 Voice Hallucination 현상
    → 학습 데이터 패턴에서 벗어날 때 발생.
    → 특히 비영어 가사, 긴 확장(Extend) 후 발생률 높음.

  원인 D: 프롬프트에 보컬 품질 지시어 없음
    → 명시하지 않으면 AI는 훈련 데이터의 평균값으로 생성.
    → "professional vocals"이 없으면 로우파이 결함도 포함.

────────────────────────────────────────────────────────────────────
■ 예방 프롬프트 (v5 적용)
────────────────────────────────────────────────────────────────────

Style Field에 추가:
  "professional studio vocals, clean and clear vocal delivery,
   pitch-perfect performance, polished and artifact-free,
   natural vocal phrasing, studio quality recording,
   no vocal glitches, smooth and consistent vocal tone"

장르를 1~2개로 제한:
  나쁜 예: "pop rock edm jazz hip-hop trap"  ← 신호 충돌
  좋은 예: "indie pop-rock"  ← 명확한 신호

가사 밀도 줄이기:
  나쁜 예: 한 줄에 20음절 이상
  좋은 예: 한 줄에 8~12음절 이내

────────────────────────────────────────────────────────────────────
■ 사후 처리 (Post-Processing)
────────────────────────────────────────────────────────────────────

  방법 1. Suno 내부 — 섹션 교체 (Replace Section)
    → 문제 있는 구간 선택 → Replace 기능으로 해당 부분만 재생성.
    → 전체를 다시 만들지 않고 부분 수정 가능.

  방법 2. 스템 분리 후 해당 트랙 수정
    → Suno Pro/Premier → 스템 내보내기 (Vocals + Instrumental 분리).
    → 문제 있는 보컬 스템에만 처리 적용.

  방법 3. iZotope RX (업계 표준 오디오 복원 툴)
    → RX의 De-click, De-crackle 기능으로 보컬 글리치 제거.
    → Spectral Repair: 아티팩트가 있는 구간을 스펙트럼상에서 수술.
    → Music Rebalance: 보컬과 악기 비율 재조정.

  방법 4. Unchirp (AI 아티팩트 전용 플러그인)
    → MP3 압축 및 AI 생성 음악 특유의 "chirp" 아티팩트 제거.
    → Suno 전용 플러그인은 아니지만 효과적.

  방법 5. 심각하면 DAW에서 Melodyne으로 수정
    → 글리치 구간의 노트를 Melodyne에서 직접 수정.

【문제 2】 과도한 리버브 (Excessive Reverb / Drowning in Reverb)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

증상:
  · 모든 것이 동굴 속에서 녹음한 것처럼 들림
  · 보컬이 리버브 꼬리에 파묻혀 선명하지 않음
  · 저음이 뭉개지고 믹스 전체가 흐릿함
  · 악기들이 서로 섞여 구분이 안 됨

────────────────────────────────────────────────────────────────────
■ 원인 분석
────────────────────────────────────────────────────────────────────

  원인 A: "Cinematic", "Epic", "Atmospheric" 등 공간감 관련 단어
    → AI가 넓은 공간 = 많은 리버브로 해석.

  원인 B: 슬로우 템포 + 감성 무드 결합
    → 느리고 감성적인 곡 = AI가 더 많은 리버브로 분위기 보상.

  원인 C: 홀/오케스트라 관련 태그
    → "Hall", "Orchestra", "Grand" 태그 = 리버브 자동 증가.

  원인 D: 리버브 제한 지시어 없음
    → 명시하지 않으면 AI는 학습 데이터 평균값 사용.

────────────────────────────────────────────────────────────────────
■ 예방 프롬프트 (v5 적용)
────────────────────────────────────────────────────────────────────

리버브 제한 표현 (Style Field에 추가):

  건조하고 선명한 믹스 원할 때:
  "dry and direct sound, minimal reverb, close-mic'd feeling,
   intimate and tight mix, no excessive reverb or delay,
   dry vocals upfront, controlled space"

  적당한 리버브 원할 때:
  "controlled reverb, subtle room ambience,
   short reverb tail, clean reverb with tight decay,
   reverb in its place, not washed out"

  적당한 홀 느낌 + 과함 방지:
  "subtle hall ambience, tasteful reverb,
   controlled wet signal, reverb not drowning the mix,
   clear and defined despite room sound"

■ 사후 처리 (Post-Processing)
────────────────────────────────────────────────────────────────────

  방법 1. Adobe Audition의 "UnSuno" 방식
    → Effect → Amplitude and Compression → Adaptive Noise Reduction.
    → 리버브 꼬리의 지속적인 배경 레이어를 노이즈로 처리.

  방법 2. iZotope RX — Reverb Reversal / Music Rebalance
    → Dialogue De-Reverb 모듈로 과도한 리버브 꼬리 줄임.
    → (보컬 스템에 적용 시 가장 효과적)

  방법 3. DAW에서 멀티밴드 컴프레션
    → 저음역 컴프레션으로 뭉개진 리버브 에너지 제어.
    → 고음역 EQ 롤오프로 리버브 꼬리의 날카로움 제거.

  방법 4. 스템 분리 후 보컬 스템에만 처리
    → 보컬 스템을 따로 추출.
    → De-Reverb 처리 후 다시 악기와 합성.

  방법 5. 트랜지언트 샤이퍼 (Transient Shaper)
    → Sustain(감쇠) 부분 줄이기 → 리버브 꼬리가 짧아짐.

【문제 3】 화이트 노이즈 / 쉬머 (White Noise / Shimmer / Hiss)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

증상:
  · 배경에서 "쉬~" 하는 지속적인 잡음
  · 고음역에서 디지털 반짝임 (Shimmer) 느낌
  · 음악 사이사이에 하이 피치 노이즈 잔재
  · 특히 조용한 구간(브릿지, 인트로)에서 더 두드러짐
  · 심벌/하이햇 주변에서 쉬머 현상 두드러짐

────────────────────────────────────────────────────────────────────
■ 원인 분석 (기술적 배경)
────────────────────────────────────────────────────────────────────

  근본 원인:
  → Suno는 "Diffusion 모델"을 사용. 화이트 노이즈 → 음악으로 정제.
  → 정제 과정에서 노이즈가 완전히 제거되지 않으면 잔재로 남음.
  → 이것이 쉬머 현상의 본질.
  → 일부 트랙은 쉬머가 적고 일부는 많음 = 생성 랜덤성.

  추가 원인:
  → 고에너지 장르 태그 → AI가 고음역을 과다 생성 → 쉬머 증가.
  → 과도한 태그 충돌 → 처리 오류 → 노이즈 잔재.

────────────────────────────────────────────────────────────────────
■ 예방 프롬프트 (v5 적용)
────────────────────────────────────────────────────────────────────

  Style Field에 추가:
  "clean and polished production, no background noise,
   pristine audio quality, studio-clean mix,
   no shimmer or digital artifacts,
   warm and smooth high frequencies, gentle top end"

  특히 장르 태그 단순화 (핵심):
  → 태그가 많을수록 쉬머가 늘어남.
  → 1~2개 핵심 장르만 명시.

  쉬머가 적은 장르 스타일:
  · Acoustic, Folk, Jazz → 쉬머 적음
  · Heavy Metal, EDM, Hyperpop → 쉬머 많음
  (고에너지 장르일수록 쉬머 발생 확률 높음)

────────────────────────────────────────────────────────────────────
■ 사후 처리 (Post-Processing)
────────────────────────────────────────────────────────────────────

  방법 1. 스템 분리 → 문제 스템 식별 (핵심!)
    → Suno의 "Get Stems" 기능.
    → 보컬 스템과 기악 스템으로 분리.
    → 쉬머는 대부분 기악(Instrumental) 스템에 있음.
    → 해당 스템만 처리 → 보컬에 영향 없이 노이즈 제거.

  방법 2. iZotope RX — Spectral De-noise
    → 쉬머가 있는 주파수 대역(보통 8~16kHz)을 분석.
    → Spectral De-noise로 지속적인 노이즈 성분 제거.
    → 리더닝(Learning) 기능: 조용한 구간을 분석해 노이즈 프로파일 생성.

  방법 3. Audacity — 노이즈 제거 (무료)
    → 조용한 구간 선택 → Effect → Noise Reduction.
    → Get Noise Profile → Effect → Noise Reduction 다시 적용.
    → 무료지만 효과적. 단, 과도하면 "뭉개짐" 발생.

  방법 4. EQ로 고음역 롤오프
    → 12kHz 이상 High Shelf를 -2 ~ -4 dB 줄임.
    → 쉬머가 있는 고음역을 약하게.
    → 단, 음악의 밝음도 함께 줄어드는 trade-off 있음.

  방법 5. denoiseai.net 같은 AI 노이즈 제거 서비스
    → Suno v4/v5에 특화된 노이즈 제거 서비스.
    → 워블, 메탈릭 꼬리, 룸 리버브 제거.
    → 단건 $1, 구독 $10/월.

  방법 6. DaVinci Resolve Fairlight (무료)
    → 스템 불러오기 → 쉬머 있는 스템에 노이즈 감소 적용.
    → EQ + 노이즈 리덕션 → 최종 합성.

증상:

  · 곡이 자연스럽게 안 끝나고 억지로 늘려진 느낌
  · 페이드아웃 대신 갑자기 뚝 끊기는 엔딩

────────────────────────────────────────────────────────────────────
■ 원인 분석
────────────────────────────────────────────────────────────────────

  원인 A: Suno가 생성 목표 길이(8분)를 채우려 억지로 연장
    → AI는 설정된 최대 길이를 채우려는 경향.
    → 음악적 내용이 소진되면 노이즈/무음으로 채움.

  원인 B: 엔딩 태그 없음
    → [Outro], [End], [Fade Out] 지시가 없으면 AI가 끝 모름.
    → 어디서 끝내야 할지 모르고 계속 생성.

  원인 C: 프롬프트에 곡 길이 지시 없음
    → 명시하지 않으면 AI가 최대 길이까지 생성 시도.

  원인 D: Extend 기능 남용
    → Extend를 반복할수록 AI가 곡의 맥락을 잃음.
    → 음악 내용 없이 빈 공간 또는 노이즈로 채움.

■ 예방 방법 (v5 적용)
────────────────────────────────────────────────────────────────────

  방법 1. Lyrics Field에 반드시 엔딩 태그 명시
  ────────────────────────────────────────────
  [Outro]
  (soft fade, strings diminuendo)
  I'll carry you...
  (fade to silence)
  [End]
  ────────────────────────────────────────────

  사용 가능한 엔딩 태그:
    [End]              → 즉시 종료 신호
    [Fade Out]         → 페이드아웃 종료
    [Outro | Fade Out] → 아웃트로 + 페이드아웃 결합
    [Final]            → 최종 섹션 표시
    (fade to silence)  → 괄호 안에 설명으로 페이드 지시

  ★ 핵심: "Suno AI는 절대 스스로 곡을 제대로 끝내지 않는다.
           반드시 [End] 또는 [Fade Out]을 직접 넣어줘야 한다."

  방법 2. Style Field에 곡 길이 명시
    "3 minute song, concise structure, natural ending"
    "2 minute 30 second track, tight composition"

  방법 3. 단계별 짧은 생성 권장
    → 전체를 한 번에 8분으로 생성하지 말 것.
    → 각 섹션을 따로 생성 → DAW에서 합치기.
    → 버스 → 코러스 → 브릿지 → 각각 독립 생성 후 스플라이싱.

  방법 1. Suno Studio — 크롭(Crop) 기능

  방법 3. DAW에서 직접 편집

    → 필요하면 페이드아웃 직접 적용.
    → 무료: Audacity / 유료: Adobe Audition, Logic, Ableton.

■ 원인 분석
────────────────────────────────────────────────────────────────────

  원인 A: 프롬프트 신호 충돌
    → 태그가 많고 서로 상충 → AI가 섹션마다 다른 해석.
    → 예: "cinematic + dry + intimate + epic" 모두 공존 = 혼란.

  원인 B: Extend 반복 시 스타일 드리프트
    → 곡을 여러 번 Extend할수록 초기 스타일에서 벗어남.
    → 각 Extend마다 새로운 랜덤 시드 → 다른 EQ 결과.

  원인 C: 복잡한 장르 혼합
    → 3개 이상 장르 = 각 섹션이 다른 장르 규칙을 적용.
    → 섹션마다 다른 믹스 전략 → 일관성 없음.

  원인 D: 참조 트랙/Persona 없음
    → 일관된 음색 기준이 없어 매 섹션 독립 생성.

■ 예방 프롬프트 (v5 적용)
────────────────────────────────────────────────────────────────────

  일관성 강화 표현 (Style Field에 추가):
  "consistent tonal balance throughout,
   cohesive mix from start to finish,
   even frequency balance, no frequency jumps,
   consistent EQ throughout all sections,
   uniform production style, same mix feel verse to chorus"

  핵심 전략 3가지:
    1. 장르 태그 1~2개로 제한.
    2. Extend 시 반드시 Callback 태그 사용:
       [Callback: keep same tonal balance and production style]
       [Callback: maintain consistent EQ and mix feel]
    3. Persona 기능 활용 (일관된 보컬/사운드 기준 저장).

  Extend 시 매번 헤더 태그 재주입:
    → 2~3회 Extend마다 Style 프롬프트를 처음부터 다시 입력.

■ 사후 처리 (Post-Processing)
────────────────────────────────────────────────────────────────────

  방법 1. DAW에서 다이나믹 EQ (Dynamic EQ)
    → FabFilter Pro-Q 3의 Dynamic EQ 모드.
    → 뭉개지는 구간의 주파수만 선택적으로 컷.
    → 일반 EQ는 항상 적용 → 다이나믹 EQ는 필요할 때만 적용.

  방법 2. 멀티밴드 컴프레서 (Multiband Compressor)
    → 각 주파수 대역을 독립적으로 컴프레션.
    → 저음이 갑자기 커지는 구간 → 저음대역 컴프로 제어.
    → iZotope Ozone Dynamics, Waves C6 등.

  방법 3. Suno Studio의 Replace 기능
    → 일관성 없는 구간을 동일 스타일로 재생성.
    → Replace 시 [Callback: same tonal balance as intro] 태그 사용.

  방법 4. iZotope Tonal Balance Control
    → 레퍼런스 트랙과 스펙트럼 비교.
    → 어떤 구간이 밸런스를 벗어나는지 시각적으로 확인.
    → 문제 구간만 집중 처리.

  방법 5. 볼륨 오토메이션
    → DAW에서 너무 큰 구간의 볼륨을 수동 낮추기.
    → 가장 원시적이지만 직관적이고 효과적.

【문제 6】 보컬 고음역 이상 (인간 음역 초과 / 귀가 아픔)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

증상:
  · 실제 인간이 절대 부를 수 없는 초고음역 보컬
  · 귀가 아플 정도로 날카롭고 찢어지는 보컬
  · 코러스에서 갑자기 소프라노 수준을 넘어서는 음역
  · "사이렌" 또는 "괴물 소리" 같은 극단적 피치
  · 보컬이 멜로딕하게 부르다가 갑자기 스크리밍 수준으로 치솟음

■ 원인 분석 (데이터 기반)
────────────────────────────────────────────────────────────────────

  원인 A: 고에너지 감정 단어가 보컬을 과열시킴
    → "Powerful", "Anthemic", "Explosive", "Epic", "High Energy"
    → "Triumphant", "Massive", "Intense", "Soaring"
    → 이런 단어들이 AI에게 "더 크고 높게 = 더 감정적"으로 해석됨.

  원인 B: 느낌표(!!) 남용 — 가장 흔한 원인
    → 가사에 !! 또는 !!! 사용 = AI에게 "최대 출력으로 질러라" 신호.
    → ! 하나만으로도 보컬이 과열되기 시작.
    → 연속 !!는 다음 줄까지 과열 상태를 유지.

  원인 C: 코러스 강도 지시 과도
    → "Belted chorus", "Screaming chorus", "Maximum power chorus"
    → AI가 이를 극단적으로 해석.

  원인 D: 특정 장르 조합
    → Metal + Opera + Epic = 극고음 발생 확률 높음.

  원인 E: 보컬 음역 지시 없음
    → 명시하지 않으면 AI가 장르 평균값(가끔 극단치) 선택.

■ 인간 보컬 음역 참고 데이터
────────────────────────────────────────────────────────────────────

  음역대 이름   │ 음역 범위         │ 최고음 기준
  ─────────────┼──────────────────┼──────────────────────
  베이스        │ E2 ~ E4           │ E4 (약 330 Hz)
  바리톤        │ G2 ~ G4           │ G4 (약 392 Hz)
  테너          │ C3 ~ C5           │ C5 (약 523 Hz)
  소프라노      │ C4 ~ C6           │ C6 (약 1047 Hz)
  메조소프라노  │ A3 ~ A5           │ A5 (약 880 Hz)
  알토          │ G3 ~ G5           │ G5 (약 784 Hz)

  → Suno가 생성하는 "이상 고음"은 C6~C7을 넘는 경우가 있음.
  → C7 이상 = 사실상 피콜로 플루트 음역 = 사람 성대로 불가능.
  → 이 구간에서 발생하는 보컬 = 디지털 왜곡 + 아티팩트 혼합.

■ 예방 프롬프트 (v5 적용) — 가장 중요한 섹션!
────────────────────────────────────────────────────────────────────

  핵심 전략: 고에너지 단어를 → 방향성 단어로 대체

  나쁜 표현 (고음 유발)      → 좋은 대체 표현
  ─────────────────────────────────────────────────────────
  "Powerful belted chorus"   → "strong chest-voice chorus, grounded"
  "Soaring vocals"           → "expressive vocals, reaching higher notes naturally"
  "Epic high notes"          → "controlled high register, tasteful high notes"
  "Anthemic, explosive"      → "anthemic but controlled, big without straining"
  "Screaming chorus"         → "passionate chorus, emotional intensity without screaming"
  "Maximum power"            → "full dynamic range, powerful but human"

  보컬 음역 직접 지정 (가장 확실한 방법):
  여성:
  "low female alto vocal, warm chest voice,
   non-soprano range, mid-register focus,
   avoid high soprano, avoid falsetto extremes,
   natural and controlled upper notes"

  남성:
  "warm tenor vocal, chest voice dominant,
   mid-range power, avoid falsetto screaming,
   controlled upper register, baritone warmth"

  보편적 음역 제한:
  "human vocal range only, no inhuman high notes,
   comfortable singing range, natural pitch delivery,
   avoid extreme high register, keep vocals earthy and grounded"

  피해야 할 단어 목록:
  ❌ Soaring   ❌ Screaming  ❌ Explosive
  ❌ Massive   ❌ Epic       ❌ Belting wildly
  ❌ Powerful!  ❌ !! (이중 느낌표)
  ❌ Ear-splitting  ❌ Glass-shattering

  ★ 가사에서 느낌표(!) 사용 지침:
    · 보통 표현: 마침표(.) 사용
    · 강조: 느낌표(!) 1개
    · 절대 !!, !!! 사용 금지
    · 강한 감정 = 느낌표가 아니라 전후 문맥과 가사 내용으로

  ★ 소리를 지르게 만드는 가사 패턴도 피하기:
    나쁜 예: "RISE UP AND SCREAM THE NAME!!!"
    좋은 예: "Rise up, call the name, you know"

  ★ 감탄사 배치 기법:
    고함 → 배경 보컬로 처리:
    주 보컬 줄: "I found my way back to you"
    배경(괄호 처리): (yeah, oh yeah)
    → 배경 보컬이 강한 감정 처리 → 주 보컬 과열 방지.

■ 사후 처리 (Post-Processing)
────────────────────────────────────────────────────────────────────

  방법 1. Suno Studio — 보컬 섹션 Replace
    → 과도한 고음이 나온 구간만 선택.
    → Replace Section + 새 프롬프트에 "controlled register" 추가.
    → 이것이 가장 직접적이고 간단한 해결책.

  방법 2. DAW — 보컬 스템 Melodyne 피치 수정
    → 스템 내보내기 → 보컬 스템만 DAW로.
    → Melodyne (가장 강력) 또는 Auto-Tune으로 피치 조정.
    → 극단적으로 높은 노트를 자연스러운 음역으로 내림.
    → Flex Pitch (Logic), Pitch Correction (Ableton)도 가능.

  방법 3. EQ De-Essing 방식 (임시방편)
    → 4,000 ~ 8,000 Hz 구간을 좁은 Q로 -4 ~ -6 dB 컷.
    → 극고음의 날카로움 줄이기.
    → 단, 음색 전체가 영향받는 단점.

  방법 4. 멀티밴드 컴프레서 고음역 제한
    → 5,000 Hz 이상 대역에 강한 컴프레션 적용.
    → 고음이 갑자기 치솟을 때 자동으로 제어.

  방법 5. 재생성이 가장 확실한 해결책
    → 위 예방 프롬프트로 재시도.
    → 대부분 3~5번 시도 안에 원하는 음역 나옴.

  원인 A: AI 컨텍스트 창 한계
    → AI는 이전 내용을 무한히 기억하지 못함.
    → 긴 생성일수록 초기 스타일 정보가 희석됨.

  원인 B: Extend 체인 오류 누적
    → Extend를 1번 할 때마다 랜덤성이 추가됨.
    → 5번 Extend = 초기와 완전히 다른 사운드 가능.

  원인 C: 프롬프트 재주입 없음
    → 초기 스타일 태그를 한 번만 적용 → 이후 구간은 무방비.

■ 예방 방법 (v5 핵심 전략)
────────────────────────────────────────────────────────────────────

  전략 1. 섹션별 분할 생성 + DAW 스플라이싱
    → 버스1(30~45초) → 생성
    → 코러스(30~45초) → 생성
    → 버스2(30~45초) → 생성
    → 각 섹션을 개별 생성 → DAW에서 붙이기.
    → 가장 안정적이고 전문가 추천 방법.

  전략 2. Extend 시 Callback 태그 필수
    → [Callback: continue with exact same style and vocal tone]
    → [Callback: maintain same mix, BPM, and vocal character]
    → [Callback: keep tempo and instrumentation consistent]
    → Extend 1~2회마다 반드시 재주입.

  전략 3. Persona 기능 활용
    → 처음 좋은 결과가 나오면 Suno Persona로 저장.
    → 이후 생성 시 이 Persona 참조 → 보컬 일관성 유지.

  전략 4. Style Influence 슬라이더 높이기 (70~85%)
    → 스타일 프롬프트를 더 강하게 따름.
    → 드리프트 감소에 직접적 효과.

────────────────────────────────────────────────────────────────────
■ 사후 처리 (Post-Processing)
────────────────────────────────────────────────────────────────────

  방법 1. 드리프트 구간 Replace
    → 품질이 떨어지기 시작하는 지점 확인.
    → Suno Studio에서 그 구간 선택 → Replace.
    → Callback 태그로 이전 스타일 기준 명시.

  방법 2. DAW에서 드리프트 구간 교체
    → 좋은 구간의 오디오를 복사 → 드리프트 구간에 붙여넣기.
    → 트랜지션 부드럽게 처리 (크로스페이드).

  방법 3. 드리프트 구간 삭제 + 페이드아웃
    → 품질이 떨어지는 구간부터 삭제.
    → 바로 전 구간에 페이드아웃 적용.
    → 더 짧지만 더 일관성 있는 곡.

【권장 워크플로우 — 전문가 수준의 완성 과정】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

단계 1. 프롬프트 작성 (품질 보호 블록 필수 포함)
  → 장르 1~2개 / 보컬 음역 명시 / 리버브 제한 / 엔딩 태그 포함.

단계 2. 30~45초 단위로 섹션별 생성
  → 버스 → 코러스 → 버스2 → 코러스 → 브릿지 → 코러스 → 아웃트로
  → 각 섹션 독립 생성 → 베스트 테이크 선택.

단계 3. Suno Studio에서 1차 편집
  → 문제 구간 Replace.
  → 엔딩 크롭 (빈 공간 삭제).
  → 페이드인/아웃 적용.

단계 4. 스템 내보내기 (Pro/Premier)
  → 보컬 + 악기 스템 분리.
  → 각 스템 개별 처리.

단계 5. DAW 후처리
  → 보컬 스템: EQ + De-esser + 리버브 조정.
  → 악기 스템: 노이즈 제거 (iZotope RX).
  → 전체 믹스버스: 글루 컴프레션.

단계 6. 마스터링
  → iZotope Ozone 또는 FabFilter Pro-L 2.
  → 장르별 LUFS 타겟 설정.
  → True Peak -1.0 dBTP 이하.

단계 7. 품질 검사
  → 여러 기기 테스트 (헤드폰, 스피커, 폰 스피커).
  → 모노 호환성 확인.
  → 레퍼런스 트랙과 A/B 비교.
