1부. 메타태그란 무엇인가? — 완전 기초 정의
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

─────────────────────────────────────────────────────────────────
■ 한 줄 정의
─────────────────────────────────────────────────────────────────

  메타태그 = 대괄호 [ ] 안에 넣는 지시어로,
  Suno AI에게 "지금 이 구간이 무엇이며 어떻게 들려야 하는지"를
  알려주는 신호 도구.

  비유: 악단에게 주는 악보의 "나타냄말"(표현 지시어)
  예: "pp (아주 약하게)", "ff (아주 강하게)", "Allegro (빠르게)"
  → 메타태그는 AI 악단에게 주는 나타냄말과 동일한 역할.

─────────────────────────────────────────────────────────────────
■ 메타태그와 일반 가사의 차이
─────────────────────────────────────────────────────────────────

  일반 가사:     그냥 노래로 불림.
  메타태그:    [ ] 안에 있으면 Suno가 "지시어"로 인식,
               노래로 부르지 않고 그 내용대로 음악을 제어함.

  예:
  [Chorus]       ← 태그: "여기가 코러스야"라고 AI에게 알림
  I love you     ← 가사: 그대로 노래로 불림

─────────────────────────────────────────────────────────────────
■ 메타태그가 실제로 하는 일 — 4가지
─────────────────────────────────────────────────────────────────

  1. 구조 제어 (Structure Control):
     어디가 버스고, 어디가 코러스인지 경계를 만들어줌.
     → "곡 지도(Map)"를 AI에게 제공.

  2. 에너지 제어 (Energy Control):
     섹션마다 에너지 레벨을 지정.
     → [Energy: High] = 코러스에서 폭발적으로.
     → [Energy: Low] = 버스에서 절제되게.

  3. 무드/분위기 제어 (Mood Control):
     감정 방향성을 제시.
     → [Mood: Melancholic] = 슬프고 사색적으로.

  4. 퍼포먼스 제어 (Performance Control):
     보컬 스타일, 악기, 효과를 지시.
     → [Vocal Style: Breathy] = 숨 섞인 보컬로.

─────────────────────────────────────────────────────────────────
■ 핵심 원리 — 태그는 "명령"이 아닌 "신호 가중치"
─────────────────────────────────────────────────────────────────

  가장 중요한 오해: "태그를 쓰면 무조건 그대로 된다" → 틀림.

  정확한 이해:
  → 태그는 확률적 가중치(signal weight).
  → AI가 그 방향으로 "더 기울어지게" 만드는 것.
  → 태그 + 가사 구조 + 스타일 필드가 함께 작동해야 최대 효과.
  → 태그만 완벽해도 가사 구조가 엉망이면 효과 감소.

  실제 검증된 성공 공식:
  강한 섹션 구조 + 명확한 태그 + 적절한 밀도 = 최상 출력

2부. 메타태그의 두 가지 종류 & 두 개의 입력 필드
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

─────────────────────────────────────────────────────────────────
■ 태그의 두 가지 종류
─────────────────────────────────────────────────────────────────

  종류 1. 섹션 태그 (Section Tags) — 가장 중요, 가장 신뢰성 높음
    → 곡의 "어디"를 정의함.
    → 예: [Intro], [Verse], [Chorus], [Bridge], [Outro]
    → 이것만 잘 써도 구조가 크게 잡힘.
    → 가장 안정적인 효과: 섹션 경계, 에너지 분리.

  종류 2. 디스크립터 태그 (Descriptor Tags) — 세부 조정용
    → 섹션 "어떻게"를 정의함.
    → 예: [Mood: Sad], [Energy: High], [Instrument: Piano]
    → 효과는 있지만 섹션 태그보다 변동성이 큼.
    → 적게 쓸수록 더 명확하게 작동함.

─────────────────────────────────────────────────────────────────
■ 두 개의 입력 필드 완전 이해
─────────────────────────────────────────────────────────────────

  Suno v5에는 프롬프트를 넣는 필드가 2개:

  ┌──────────────────────────────────────────────────────────────┐
  │ 필드 1: Style Field (스타일 필드)                              │
  │ → [ ] 대괄호 없이 자연어로 작성                                │
  │ → 곡 전체의 큰 그림: 장르, 무드, 악기, 보컬 성별, BPM 등       │
  │ → 여기서 쓴 내용은 곡 전체에 적용됨                             │
  │ → 태그 형식 사용 X, 설명적 단어나 쉼표 구분 사용               │
  │                                                               │
  │ 예:                                                           │
  │ "K-pop, 120 BPM, C major, warm piano, ethereal female vocals, │
  │  melancholic but hopeful mood, wide stereo, radio-ready"      │
  └──────────────────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────────────┐
  │ 필드 2: Lyrics Field (가사 필드)                               │
  │ → 가사 + 메타태그를 함께 넣는 곳                               │
  │ → [ ] 대괄호 형식으로 태그 사용                                 │
  │ → 섹션 구조를 정의하고 세부 지시를 담음                         │
  │ → 여기가 메타태그를 쓰는 주 공간                               │
  │                                                               │
  │ 예:                                                           │
  │ [Mood: Longing]                                               │
  │ [Energy: Low]                                                 │
  │ [Intro]                                                       │
  │ (soft piano opens, space and silence)                         │
  │                                                               │
  │ [Verse 1]                                                     │
  │ 그 날 빗소리가 아직도 들려                                      │
  │ 창문 너머 네 그림자 같은 게                                     │
  └──────────────────────────────────────────────────────────────┘

  ⚠️ 핵심 원칙: Style Field = 큰 그림 / Lyrics Field = 세부 지시
  두 필드가 서로 충돌하면 안 됨!

카테고리 A. 섹션 구조 태그 (Section Structure Tags)
— 가장 기본, 가장 중요, 신뢰성 최고 —
════════════════════════════════════════════════════════════════════

  기본 섹션 태그:
  ┌─────────────────┬──────────────────────────────────────────┐
  │ 태그              │ 역할 & 설명                               │
  ├─────────────────┼──────────────────────────────────────────┤
  │ [Intro]          │ 도입부. 팔레트 설정, 짧게 유지             │
  │ [Verse]          │ 버스. 이야기 전개. 코러스보다 에너지 낮음   │
  │ [Verse 1]        │ 1절 버스 (번호 지정으로 구분 명확화)        │
  │ [Verse 2]        │ 2절 버스 (새 각도, 같은 그루브)            │
  │ [Pre-Chorus]     │ 프리코러스. 기대감 상승. 짧은 문장 권장    │
  │ [Chorus]         │ 코러스. 훅 구간. 최고 에너지               │
  │ [Post-Chorus]    │ 코러스 이후 여운 구간                      │
  │ [Bridge]         │ 브릿지. 대비 섹션. 무드·리듬 변화          │
  │ [Final Chorus]   │ 마지막 코러스. 가장 큰 버전                │
  │ [Outro]          │ 아웃트로. 착지, 페이드아웃 준비             │
  │ [End]            │ 곡 완전 종료 신호                          │
  │ [Hook]           │ 훅 구간 (코러스와 별개로 쓸 수 있음)        │
  │ [Refrain]        │ 리프레인 (반복 후렴)                       │
  │ [Break]          │ 짧은 휴지 구간                             │
  │ [Interlude]      │ 간주 (기악 연주 구간)                      │
  │ [Instrumental]   │ 기악 연주 전용 구간 (보컬 없음)             │
  │ [Solo]           │ 솔로 연주 구간                             │
  │ [Intro | Fade In]│ 페이드인으로 시작하는 인트로                │
  │ [Outro | Fade Out]│ 페이드아웃으로 끝나는 아웃트로             │
  └─────────────────┴──────────────────────────────────────────┘

  에너지 동작 태그:
  ┌──────────────────┬─────────────────────────────────────────┐
  │ 태그               │ 역할                                     │
  ├──────────────────┼─────────────────────────────────────────┤
  │ [Build]           │ 긴장감 상승 구간 (코러스 전에 사용)        │
  │ [Build-Up]        │ Build보다 더 점층적 상승                  │
  │ [Drop]            │ 강한 임팩트 구간 (EDM 드롭 등)             │
  │ [Breakdown]       │ 밀도 낮추기, 대비 공간 만들기              │
  │ [Climax]          │ 곡의 최고 정점                            │
  │ [Release]         │ 긴장 해소 구간                            │
  └──────────────────┴─────────────────────────────────────────┘

카테고리 B. 무드·감정 태그 (Mood & Emotion Tags)
— [Mood: ___] 형식으로 사용 —
════════════════════════════════════════════════════════════════════

  긍정 감정:
  [Mood: Uplifting]       → 고양되고 희망적
  [Mood: Euphoric]        → 극도의 행복감
  [Mood: Joyful]          → 기쁨
  [Mood: Playful]         → 장난스럽고 경쾌함
  [Mood: Optimistic]      → 낙관적
  [Mood: Confident]       → 당당하고 자신 있음
  [Mood: Triumphant]      → 승리감, 개선
  [Mood: Hopeful]         → 희망적

  슬픔·그리움:
  [Mood: Melancholic]     → 우울하고 사색적
  [Mood: Longing]         → 그리움, 애틋함
  [Mood: Nostalgic]       → 추억을 그리워함
  [Mood: Bittersweet]     → 씁쓸하지만 달콤한 감정
  [Mood: Somber]          → 음울하고 심각함
  [Mood: Heartbroken]     → 상실감, 이별의 아픔
  [Mood: Wistful]         → 아련하고 아쉬운

  긴장·어두운:
  [Mood: Tense]           → 긴장감
  [Mood: Haunting]        → 귀신 같은, 잊히지 않는
  [Mood: Dark]            → 어두움
  [Mood: Mysterious]      → 신비롭고 불확실함
  [Mood: Eerie]           → 으스스함
  [Mood: Ominous]         → 불길함

  차분한·사색적:
  [Mood: Calm]            → 평온함
  [Mood: Serene]          → 고요하고 맑음
  [Mood: Meditative]      → 명상적
  [Mood: Introspective]   → 내면을 들여다보는
  [Mood: Contemplative]   → 사색적
  [Mood: Peaceful]        → 평화로운

  강렬·열정:
  [Mood: Passionate]      → 열정적
  [Mood: Intense]         → 강렬함
  [Mood: Energetic]       → 활기찬
  [Mood: Aggressive]      → 공격적, 강한
  [Mood: Anthemic]        → 웅장하고 함성 같은
  [Mood: Raw]             → 날 것의 감정 (가공하지 않은)
  [Mood: Urgent]          → 긴박함

  로맨틱:
  [Mood: Romantic]        → 로맨틱
  [Mood: Sensual]         → 관능적 (과도한 보컬 억제 효과)
  [Mood: Tender]          → 부드럽고 다정함
  [Mood: Flirtatious]     → 설레고 장난기 있는 연애 감정

카테고리 C. 에너지 레벨 태그 (Energy Level Tags)
— [Energy: ___] 형식으로 사용 —
════════════════════════════════════════════════════════════════════

  [Energy: Low]           → 조용하고 절제됨 (버스, 인트로)
  [Energy: Medium]        → 중간 수준 (버스→코러스 전환)
  [Energy: High]          → 강렬하고 폭발적 (코러스, 파이널 코러스)
  [Energy: Building]      → 점점 상승 중
  [Energy: Dropping]      → 갑자기 줄어드는
  [Energy: Medium→High]   → 중간에서 높음으로 전환
  [Energy: Restrained]    → 억제된, 참고 있는 에너지

  ⭐ 핵심 팁:
  → [Energy: High]는 코러스 바로 앞 줄에 배치해야 효과 최대.
  → 곡 상단에만 쓰면 "전체가 너무 시끄러운" 결과가 됨.
  → 로컬 배치 (섹션 바로 위) = 해당 섹션에만 적용.

카테고리 D. 보컬 스타일 태그 (Vocal Style Tags)
— [Vocal Style: ___] 또는 단독 태그로 사용 —
════════════════════════════════════════════════════════════════════

  성별·음역:
  [Male Vocal]            → 남성 보컬
  [Female Vocal]          → 여성 보컬
  [Duet]                  → 남녀 듀엣
  [Soprano]               → 소프라노 (고음 여성)
  [Alto]                  → 알토 (중저음 여성)
  [Tenor]                 → 테너 (고음 남성)
  [Baritone]              → 바리톤 (저음 남성)
  [Bass Vocal]            → 베이스 보컬 (최저음 남성)

  창법 스타일:
  [Vocal Style: Breathy]  → 숨 섞인 부드러운 창법
  [Vocal Style: Belt]     → 강하고 파워풀한 발성
  [Vocal Style: Falsetto] → 팔세토 (헤드 보이스)
  [Vocal Style: Raspy]    → 거칠고 쉰 목소리
  [Vocal Style: Smooth]   → 부드럽고 매끄러운
  [Vocal Style: Powerful] → 강렬하고 힘 있는
  [Vocal Style: Warm]     → 따뜻하고 아늑한 음색
  [Vocal Style: Close-Mic]→ 귀에 가까운 친밀한 느낌
  [Vocal Style: Ethereal] → 몽환적이고 신비로운
  [Vocal Style: Gospel]   → 가스펠 느낌의 강렬한 감정

  화음·구성:
  [Harmonies]             → 화음 추가
  [Stacked Harmonies]     → 여러 겹 화음 (풍성한 코러스 느낌)
  [Call and Response]     → 주고받기 형식
  [Ad-lib]                → 즉흥 보컬 애드리브
  [Background Vocals]     → 배경 보컬
  [Choir]                 → 합창

  특수 보컬:
  [Spoken Word]           → 노래 아닌 낭독
  [Narration]             → 나레이션
  [Rap]                   → 랩
  [Whisper]               → 속삭임
  [Chanting]              → 챈팅 (주문 외듯 반복)
  [Scat]                  → 재즈의 스캣 창법
  [Melisma]               → 멜리스마 (한 음절에 여러 음)

카테고리 E. 악기 태그 (Instrument Tags)
— [Instrument: ___] 형식으로 사용 —
════════════════════════════════════════════════════════════════════

  현악기:
  [Instrument: Piano]           → 피아노
  [Instrument: Electric Piano]  → 일렉트릭 피아노 (로즈 등)
  [Instrument: Acoustic Guitar] → 어쿠스틱 기타
  [Instrument: Electric Guitar] → 일렉트릭 기타
  [Instrument: Bass Guitar]     → 베이스 기타
  [Instrument: Violin]          → 바이올린
  [Instrument: Cello]           → 첼로
  [Instrument: Strings]         → 현악 섹션 전체

  건반:

4부. 메타태그 배치 규칙 — "어디에" 쓰느냐가 핵심
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

─────────────────────────────────────────────────────────────────
■ 배치의 두 가지 방식
─────────────────────────────────────────────────────────────────

  전역 배치 (Global Placement) — 가사 필드 상단:
    → 곡 전체에 걸쳐 영향을 미침.
    → 큰 그림 지시에 사용.
    → 예:
      [Mood: Melancholic]
      [Energy: Medium]
      [Instrument: Piano, Soft Strings]
      (이후 가사 시작)

  로컬 배치 (Local Placement) — 특정 섹션 바로 위:
    → 해당 섹션에만 적용됨.
    → 에너지 전환에 특히 효과적.
    → 예:
      [Verse]
      조용한 아침이야
      아무것도 없어

      [Chorus]          ← 이 태그는 코러스에만 적용
      [Energy: High]    ← 코러스에만 에너지 올림
      이제 날아오를 거야

─────────────────────────────────────────────────────────────────
■ 배치 5대 원칙 (검증된 규칙)
─────────────────────────────────────────────────────────────────

  원칙 1. 팔레트 탑로드 (Top-Load the Palette):
    → 가사 필드 맨 위에 핵심 무드 1개 + 에너지 방향 1개 +
      앵커 악기 1~2개를 먼저 배치.
    → 이유: 초반 편곡 설정에 가장 큰 영향.

    예:
    [Mood: Hopeful]
    [Energy: Medium]
    [Instrument: Piano, Warm Bass]
    [Female Vocal] [Breathy]

  원칙 2. 핵심 전환점은 로컬 배치:
    → "여기서 터진다"는 구간 바로 위에 [Energy: High] 배치.
    → 전역에만 쓰면 곡 전체가 시끄러워짐.

  원칙 3. 태그 1개 = 1가지 임무:
    → [Mood: sad, happy] ← 충돌 = 효과 감소
    → [Mood: Bittersweet] ← 단일 복합 감정 = 효과 최대

  원칙 4. 악기는 2~3개만:
    → [Instrument: Piano, Guitar, Strings, Drums, Synth] ← 너무 많음
    → [Instrument: Piano, Soft Drums] ← 적절

  원칙 5. 섹션 태그 → 디스크립터 태그 → 가사 순서:
    올바른 순서:
    [Chorus]          ← 섹션 태그 먼저
    [Energy: High]    ← 디스크립터 태그 다음
    가사 내용         ← 가사 마지막

■ 무드 일치의 핵심 원리 — "4중 일치 법칙"
─────────────────────────────────────────────────────────────────

  빌보드급 히트곡 무드 제어의 핵심:
  무드 태그 + 에너지 태그 + 악기 태그 + 가사 내용
  이 4가지가 모두 같은 방향을 가리켜야 한다.

  예시: "이별의 슬픔" 무드

  ✅ 4중 일치 (올바른 방법):
    [Mood: Heartbroken]      → 감정 태그: 상실
    [Energy: Low]            → 에너지 태그: 절제
    [Instrument: Piano]      → 악기 태그: 슬픈 피아노
    "네 번호를 지우지 못했어"  → 가사: 구체적 이별 장면

  ❌ 4중 충돌 (잘못된 방법):
    [Mood: Heartbroken]      → 슬픔
    [Energy: High]           → 폭발적 ← 슬픔과 충돌
    [Instrument: Brass]      → 밝고 강함 ← 슬픔과 충돌
    "너를 사랑해 영원히"      → 사랑 고백 ← 이별과 충돌

■ 무드 이동 설계 — 섹션마다 무드가 달라야 하는 이유

  예: 이별 발라드의 무드 이동
  인트로:      [Mood: Calm]         → 고요한 시작
  버스 1:      [Mood: Melancholic]  → 슬픔 진입
  프리코러스:  [Mood: Longing]      → 그리움 고조
  코러스:      [Mood: Heartbroken]  → 이별 폭발
  버스 2:      [Mood: Wistful]      → 아련한 회상
  코러스:      [Mood: Heartbroken]  → 재확인
  브릿지:      [Mood: Desperate]    → 절박함 피크
  파이널 코러스:[Mood: Bittersweet] → 받아들임
  아웃트로:    [Mood: Serene]       → 평화로운 해소

  이것이 청중이 "눈물 나고 카타르시스 느끼는" 이유.
  무드가 처음부터 끝까지 똑같으면 → 감동 없음.

팁 그룹 1. 배치 팁 (Placement Tips)
══════════════════════════════════════════════════════════════════

  팁 1. 코러스 에너지는 반드시 코러스 바로 앞에
    → 곡 상단에 [Energy: High] 하나만 쓰면 전체가 시끄러워짐.
    → [Chorus] 바로 아래 줄에 [Energy: High] 배치 = 효과 극대화.

  팁 2. 브릿지는 무조건 대비 설계
    → [Bridge] 안 가사가 버스처럼 들리면 → 브릿지 실패.
    → [Breakdown] 태그 + 다른 [Mood: ___] + 짧은 라인 = 올바른 브릿지.

  팁 3. 아웃트로는 항상 여백을 남겨라
    → 마지막 가사 이후 (fade to silence) + [End] 배치.
    → 이것 없으면 Suno가 랜덤 소리로 끝을 채움.

  팁 4. 인트로는 태그만 쓰고 가사 없애기
    → [Intro] 아래 (soft piano opening, minimal) 정도만.
    → 가사 넣으면 인트로가 너무 빨리 시작됨.

  팁 5. 섹션 사이 빈 줄 필수
    → [Chorus] 이후 [Verse 2] 전에 반드시 빈 줄 한 칸.
    → 붙여쓰면 섹션 경계를 AI가 놓침.

══════════════════════════════════════════════════════════════════
팁 그룹 2. 태그 밀도 팁 (Tag Density Tips)
══════════════════════════════════════════════════════════════════

  팁 6. "태그 많음 = 좋음" 이 아님
    → 태그 10개 > 태그 3개가 더 나쁜 결과 낼 때 많음.
    → 섹션 태그 + 핵심 디스크립터 2~3개 = 최적.
    → "더 적게, 더 명확하게"가 황금 원칙.

  팁 7. 상충하는 태그는 독약
    → [Mood: Sad] + [Energy: High] = AI가 둘 다 평균냄 → 애매한 결과.
    → [Mood: Bittersweet] + [Energy: Medium] = 단일 명확 신호 → 좋은 결과.

  팁 8. 악기 태그는 2~3개가 한계
    → [Instrument: Piano, Strings] = 선명한 팔레트.
    → [Instrument: Piano, Guitar, Strings, Synth, Brass, Drums] = 혼돈.

  팁 9. 보컬 태그는 스타일 1~2개만
    → [Breathy] + [Belt] = 충돌.
    → [Warm] + [Close-Mic] = 조화로운 조합.

  팁 10. 섹션 태그가 먼저, 디스크립터는 그 아래
    → 올바른 순서:
      [Chorus]
      [Energy: High]
      [Stacked Harmonies]
      (가사)
    → 잘못된 순서:
      [Energy: High]
      [Stacked Harmonies]
      [Chorus]
      (가사)

══════════════════════════════════════════════════════════════════
팁 그룹 3. 가사 작성 팁 (Lyric Engineering Tips)
══════════════════════════════════════════════════════════════════

  팁 11. 소괄호 = 퍼포먼스 지시, 대괄호 = 구조 지시
    → (breathe in) = 보컬이 숨 들이쉬는 표현
    → (ohh, yeah~) = 보컬 애드리브
    → [Breath] = 잘못된 사용법

  팁 12. 코러스 가사는 최대한 짧게
    → 코러스 한 줄 = 8~12음절이 이상적.
    → 길면 훅이 기억에 안 남음.
    → 예: "사랑했어 정말로" (6음절) ← 이상적
            "나는 너를 정말로 사랑했는데 넌 몰랐어" ← 너무 김

  팁 13. 버스 2는 버스 1과 달라야 함
    → 같은 무드, 같은 리듬이지만 새로운 각도의 이야기.
    → "왜" → 버스 1 / "그래서 어떻게" → 버스 2 구조가 좋음.

  팁 14. 발음으로 창법 제어
    → "loooove" = 멜리스마 유도
    → "(breath) I never said" = 숨 이후 가사 시작
    → "seen, seen!" = 강조 반복

  팁 15. 브릿지 가사는 문장을 짧게, 리듬을 바꿔라
    → 버스와 코러스가 4박자 리듬이면 브릿지는 3박 느낌으로.
    → 줄 길이도 다르게.

══════════════════════════════════════════════════════════════════
팁 그룹 4. v5 고급 팁 (v5 Advanced Tips)
══════════════════════════════════════════════════════════════════

  팁 16. Extend 시 Callback 태그 필수
    → 곡을 연장할 때 [Callback: maintain exact same style,
      vocal tone, and BPM as the original] 항상 포함.
    → 이것 없으면 스타일 드리프트 발생 (품질 저하).

  팁 17. 파이널 코러스 전 [Key Change Up]
    → 반음 전조 = 청중에게 "와!" 효과.
    → [Final Chorus] 태그와 함께 [Key Change Up] 같이 배치.

  팁 18. 루프 콘텐츠용 [Structure: seamless loop]
    → 틱톡, 릴스용 음악은 이 태그로 끝이 처음과 연결.
    → (loop-friendly) 주석도 함께 쓰면 효과 강화.

  팁 19. Replace + Callback으로 특정 구간 교체
    → 마음에 안 드는 구간을 Replace 기능으로 재생성.
    → [Callback: keep same energy, instruments, and vocal character
       as the rest of the song] 추가로 일관성 유지.

  팁 20. 생성 후 Studio에서 마무리
    → 태그가 아무리 좋아도 완성 직후 바로 최종 출력이 아님.
    → Suno Studio에서:
      · Fade in/out 수동 조절
      · 불필요한 구간 Region 편집
      · BPM 고정 후 DAW export
    → 이 단계가 있어야 "진짜 완성곡"이 됨.

7부. 자주 나타나는 문제 & 해결법
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  문제 1: 코러스가 버스보다 더 크게 안 들림
    원인: [Energy: High] 전역 배치 / 코러스 가사가 너무 김
    해결: [Chorus] 바로 아래 [Energy: High] 로컬 배치 +
          가사 줄이기 + [Build-Up] 프리코러스에 추가

  문제 2: 태그가 노래로 불려버림
    원인: 대괄호 형식 오류, 또는 Suno가 태그를 인식 못함
    해결: [Tag] 형식 확인, 태그가 가사 줄과 분리되어 있는지 확인

  문제 3: 브릿지가 버스처럼 들림
    원인: 브릿지 가사 구조가 버스와 동일
    해결: [Bridge] + [Breakdown] 태그 조합 +
          [Mood: ___] 브릿지 전용 무드 + 짧은 가사 줄

  문제 4: 곡 끝에 이상한 소리가 남
    원인: 엔딩 신호 없음
    해결: [Outro] + (fade to silence) + [End] 항상 마지막에

  문제 5: 보컬이 너무 높거나 귀가 아픔
    원인: "Soaring", "Explosive", "!!" 등 과장 표현
    해결: [Vocal Style: Warm] + [Human vocal range only] +
          감탄 표현 제거

  문제 6: Extend 후 스타일이 달라짐
    원인: Callback 태그 없음
    해결: Extend 프롬프트에 [Callback: same style, vocal tone,
          BPM as original] 추가

  문제 7: 전체 곡이 에너지 없이 밋밋함
    원인: Style Field에 "soft, gentle, quiet" 만 쓰고 대비 없음
    해결: 버스 = 낮은 에너지 / 코러스 = 높은 에너지 대비 설계

9부. 메타태그를 사용하기 위해 반드시 알아야 할 요소 정리
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  총 6개 레이어가 메타태그 효과를 결정한다:

  레이어 1. Style Field 설계:
    → 장르·BPM·키·무드·악기·보컬 큰 그림.
    → 여기가 틀리면 태그 아무리 잘 써도 방향이 어긋남.

  레이어 2. 메타태그 (본 가이드 내용):
    → 섹션 경계·에너지·무드·보컬·악기 세부 제어.
    → 신호 가중치. 명령이 아님.

  레이어 3. 가사 구조 설계:
    → 코러스가 코러스답게 쓰여야 코러스로 인식됨.
    → 브릿지가 버스와 달라야 대비가 생김.
    → 가사가 태그를 지지해야 효과 최대.

  레이어 4. 감정 집중 (Emotional Focus):
    → 하나의 핵심 감정으로 곡 전체를 일관성 있게 유지.
    → 여러 감정 방향이 충돌하면 평균화 = 밋밋함.

  레이어 5. 편곡 절제 (Arrangement Discipline):
    → 악기 적게 = 선명함.
    → 악기 많음 = 뭉침.
    → 빈 공간도 편곡의 일부.

  레이어 6. Studio 마무리:
    → Fade in/out, 지역 편집, BPM 고정, 내보내기.
    → 태그가 아무리 좋아도 Studio 없이는 미완성.
